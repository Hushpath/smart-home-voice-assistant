## 4 测试策略

1. 后端使用 pytest + FastAPI TestClient + 临时 SQLite，避免污染本地 `backend/data/app.db`。
2. 测试中禁用真实天气外部请求，验证 Open-Meteo 成功分支时使用 monkeypatch，失败分支验证本地备用数据。
3. 语音测试使用 `MockASRProvider`，仅作为测试和本地开发验证，不写成生产能力。
4. 前端通过 `npm run build` 验证构建可用性。
5. 页面通过本地后端和前端服务登录默认账号后截图。

### 6.2 TC-001 用户注册、登录和当前用户

前置条件：测试库为空或使用隔离 TestClient。测试数据：新的用户名、密码和昵称。测试步骤：调用注册接口，检查密码不以明文落库；调用登录接口，检查返回 `access_token`；携带 token 调用 `/api/auth/me`。预期结果：注册成功、登录成功、当前用户与注册用户一致。实际结果：由 `test_auth_and_devices.py` 覆盖并在本次全量 pytest 中通过。证据：`docs/images/pytest_result.txt`。

### 6.3 TC-002 鉴权边界

前置条件：未携带 token 或 token 无效。测试步骤：访问房间、设备、语音、个性化等受保护接口。预期结果：返回未授权错误，不泄露业务数据。实际结果：认证、语音、个性化测试均包含未授权访问验证，本次通过。该用例验证 NFR-001，但不等同于安全渗透测试。

### 6.4 TC-003 至 TC-005 设备与 Dashboard

测试数据来自初始化脚本中的默认家庭、4 个房间和 20 个虚拟设备。测试步骤包括：登录后查询 Dashboard，读取房间和设备列表，按房间筛选设备，修改设备开关或属性，再查询设备详情和历史。预期结果：统计数据非空，设备状态变化持久化，历史记录包含变更前后状态和来源。实际结果：`test_auth_and_devices.py`、`test_command_execute.py` 和 smoke test 覆盖该流程，本次通过。

### 6.5 TC-006 至 TC-010 指令解析、执行和多指令

测试数据包括开灯、关电视、调温度、调亮度、查状态、创建提醒、天气查询、场景执行和多设备/多房间表达。测试步骤：先调用 `/api/commands/parse` 校验 intent、room、device_type、target_state、confidence 等字段，再调用 `/api/commands/execute` 校验虚拟设备状态、错误处理、批量结果和日志。预期结果：合法指令被解析执行，越界参数和歧义指令不会错误修改设备，多指令允许部分成功并记录失败子项。实际结果：`test_command_parse.py`、`test_command_execute.py` 和 `test_multi_command.py` 覆盖并通过。

### 6.6 TC-011 至 TC-012 方言归一和 ASR 后纠错

测试数据包含粤语、西南口语、东北/北方口语、ASR 错词和领域词表纠错样例。测试步骤：直接测试 normalizer/corrector 输出，也通过指令解析和日志验证纠错信息是否进入链路。预期结果：常见表达可被转换为可解析文本，纠错 trace 能在日志中解释。实际结果：`test_dialect_normalizer.py` 和 `test_asr_post_corrector.py` 通过。限制：该测试不覆盖真实方言语音识别准确率。

### 6.7 TC-013 至 TC-015 语音链路和 ASR 配置

测试数据使用 Mock ASR 音频和 ASR 配置字段。测试步骤：查询 Provider 状态，保存/读取/删除 ASR 配置，调用 Mock 识别和语音执行，验证 cloud/xunfei provider 的超时、认证、响应格式错误。预期结果：Mock 路径可验证后端链路；配置接口返回脱敏信息；云端错误清晰返回。实际结果：`test_voice.py`、`test_cloud_asr_provider.py` 和 smoke test 通过。限制：未使用真实讯飞密钥联调，不能写成真实云端 ASR 已通过。

### 6.8 TC-016 日志详情

测试步骤：执行文本指令和语音指令后查询 `/api/commands/logs`，检查日志摘要、原始输入、ASR 信息、纠错、归一、解析、执行、批量结果和 raw 详情。预期结果：日志能支持课程材料中“可追溯”的说明。实际结果：`test_command_logs_detail.py` 通过；页面截图 `docs/images/command_logs.png` 显示日志页可访问。

### 6.9 TC-017 至 TC-019 提醒、天气和场景

提醒测试覆盖创建、查询、修改、完成状态和删除；天气测试覆盖公开接口和 fallback；场景测试覆盖列表和执行动作。预期结果：提醒数据按用户保存，天气返回 source，场景执行修改虚拟设备状态。实际结果：相关用例由 `test_command_execute.py` 和 smoke test 覆盖并通过。限制：提醒不包含后台通知；天气 fallback 不等同于真实天气服务 SLA。

### 6.10 TC-020 至 TC-022 个性化、别名和常用指令

测试步骤：读取默认偏好，更新默认方言和输入方式；为设备创建别名，验证重复别名拒绝、删除和指令命中；执行成功指令后读取常用指令和学习建议。预期结果：个性化数据与当前用户关联，别名能被指令执行链路使用。实际结果：`test_personalization.py` 通过。本次进一步补充 `test_user_data_isolation.py`，验证不同用户之间的日志、提醒和设备别名不可见。

### 6.11 TC-023 至 TC-025 构建、smoke 和截图

测试步骤：在 `frontend` 执行 `npm run build`；在 `backend` 执行 `scripts/smoke_test.py`；访问本地页面并生成截图。预期结果：构建成功、smoke test 全部 `[OK]`、主要页面可截图。实际结果：前端构建成功但存在大 chunk 和第三方注释警告；smoke test 通过；登录、Dashboard、语音、设备、日志、提醒和场景截图已生成。证据见 `docs/images/frontend_build.txt`、`docs/images/smoke_test_result.txt` 和 `docs/images/*.png`。
