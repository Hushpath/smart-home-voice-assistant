# 智能家居语音交互助手系统测试计划与用例报告

## 1 引言

### 1.1 文档目的

本文档说明智能家居语音交互助手当前交付版本的测试计划、测试用例、执行结果、缺陷限制和测试结论。文档结论仅基于已执行的自动化测试、smoke test、前端构建、已有截图证据和源码核验；未执行或证据不足的内容均单独标注。

### 1.2 测试对象

测试对象包括 FastAPI 后端、SQLite 数据模型、鉴权与统一响应、设备与状态历史、中文指令解析与执行、命令日志、提醒天气场景、个性化配置、Voice 与 ASR 配置接口、MockASRProvider 路径、浏览器识别文本和手工文本兜底路径，以及 Vue 3 前端构建结果。


## 2 测试目标

### 2.1 功能正确性目标

验证 FR-001 至 FR-008 中的用户认证、家庭概览、设备管理、指令理解、语音输入、日志追踪、提醒天气场景和个性化能力是否在当前实现和测试范围内形成闭环。

### 2.2 接口稳定性目标

验证 API-001 至 API-033 中当前真实存在的接口在正常、异常和未授权场景下返回统一响应结构，并能通过 pytest 或 smoke test 被重复执行。

### 2.3 语音输入与兜底链路目标

分别验证云端 ASR 配置状态、MockASRProvider 测试路径、云端未配置或失败时的 fallback 返回、浏览器识别文本进入 `/api/commands/execute` 的设计路径、手工文本输入执行路径，并明确真实云端 ASR 成功识别未在本轮执行。

### 2.4 权限与安全边界目标

验证登录、错误密码、未携带 token、无效 token、受保护接口拒绝访问、密码非明文保存、ASR 配置脱敏返回等基础安全边界。该目标不等同于安全渗透或攻击专项。

### 2.5 前端构建与页面可验证性目标

验证 `npm run build` 能完成生产构建，主要页面和组件可以进入构建产物。已有页面截图作为页面状态证据，但本轮未重新执行浏览器手工路径和多浏览器矩阵。

### 2.6 日志可解释性目标

验证一次指令执行可以在 `command_logs` 中追踪 `trace_id`、输入来源、ASR 信息、识别文本后纠错、方言归一、解析、执行、批量结果和个性化命中。

### 2.7 非功能需求验证目标

在当前本地验证环境中验证 NFR-001 至 NFR-008 的基础可测项，同时明确性能、兼容性、安全和生产部署等专项未执行对质量结论的限制。

## 6 测试策略

### 6.1 后端自动化测试策略

使用 pytest 和 FastAPI TestClient 对后端接口、服务层和业务流程进行回归验证。每个测试运行在临时 SQLite 数据库上，降低对本地持久化数据的影响。证据以 `docs/images/pytest_result.txt` 为准。

### 6.2 接口测试策略

接口测试覆盖公开接口、受保护接口、统一响应格式、参数校验、未授权访问和业务错误码。主要通过 `client.get/post/patch/delete` 直接调用 `/api/*` 路径，避免绕过路由层。

### 6.3 业务流程测试策略

业务流程测试围绕登录后进入家庭数据、控制设备、执行指令、写日志、查历史、管理提醒、查询天气、执行场景和应用个性化展开。Smoke test 验证主链路是否能在临时数据库中闭合。

### 6.4 异常输入测试策略

异常输入覆盖空指令、未知指令、参数越界、枚举非法、不存在房间、不存在设备、不支持操作、低置信度状态改变类指令、云端 ASR 未配置、空音频、缺失音频和不支持音频格式。

### 6.5 鉴权边界测试策略

鉴权测试覆盖错误密码、未携带 token、无效 token、受保护接口拒绝访问、用户数据隔离和 ASR 配置接口鉴权。该策略用于验证基础访问控制，不替代安全渗透专项。

### 6.6 前端构建测试策略

前端构建使用 `npm run build` 验证 Vue 3、Vite、Element Plus、Pinia、路由、页面和组件可被生产构建。构建成功不等同于所有页面交互已在浏览器中手工验证。

### 6.7 页面 smoke 或手工验证策略

当前引用 `docs/images/` 中已有页面截图作为页面状态证据。本轮未重新启动浏览器手工执行登录、导航、语音、设备、日志、提醒和场景全流程，因此页面手工验证结论标记为待补充验证。

### 6.8 云端 ASR、Mock 与兜底测试策略

云端 ASR 相关验证分为五类：配置状态与脱敏、MockASRProvider 测试路径、云端未配置或失败 fallback、Fake provider 模拟通用 cloud/Xunfei 分支、真实外部 ASR 成功识别。前四类已有自动化证据，最后一类本轮未执行。

### 6.9 回归测试策略

本轮新增两条鉴权测试后，先执行 `tests/test_auth_and_devices.py`，再执行全量 pytest，并以全量 187 passed 作为最终回归证据。

## 11 测试结论

### 11.1 功能正确性结论

在已执行的后端自动化测试、smoke test 和前端构建范围内，FR-001 至 FR-008 均有测试用例承接。后端全量 pytest 187 passed，说明认证、设备、指令、日志、语音接口、提醒、天气、场景、个性化和用户隔离在当前测试数据下满足核心功能验证要求。

### 11.2 接口稳定性结论

API-001 至 API-033 中当前真实存在的接口均能在设计对象到测试关注点矩阵中找到测试承接。自动化测试覆盖正常、异常和未授权场景，统一响应格式也由认证测试和异常测试覆盖。

### 11.3 语音链路结论

MockASRProvider、ASR 配置接口、云端未配置或失败 fallback、通用 cloud provider 模拟分支和 Xunfei 模拟分支均有自动化证据。真实外部 ASR 成功识别、真实准确率、复杂噪声环境和多厂商完整兼容未执行，不能写成已通过。

### 11.4 权限与安全边界结论

基础鉴权、错误密码、未携带 token、无效 token、用户数据隔离和 ASR 配置脱敏已通过自动化验证。该结论只覆盖基础访问控制，不覆盖安全渗透、XSS、SQL 注入、CORS 生产策略和完整权限模型。

### 11.5 前端构建与页面可验证性结论

前端生产构建成功，说明 UI-001 至 UI-007 相关页面和组件能通过 Vite 构建。已有页面截图可作为页面状态证据，但本轮未重新执行浏览器手工流程，浏览器 Web Speech 实机识别和多浏览器页面验证仍需补充。

### 11.6 日志与可追踪性结论

日志摘要、日志详情、`trace_id`、ASR、识别文本后纠错、方言归一、解析、执行、批量和个性化命中均由自动化测试覆盖。日志当前用于交互解释和工程定位，不等同生产级审计系统。

### 11.7 个性化能力结论

默认偏好、设备别名、常用指令、学习建议、别名参与指令解析和跨用户隔离已通过自动化测试。当前个性化基于规则和频次统计，不涉及用户专属模型训练。

### 11.8 非功能需求结论

NFR-001 至 NFR-005、NFR-008 在基础层面有自动化或文档追踪证据；NFR-006 性能边界和 NFR-007 兼容性仅有构建、smoke 和边界说明，缺少压力、多浏览器和移动端专项。当前非功能结论仅覆盖本地验证环境和当前测试范围。

### 11.9 总体结论

当前交付版本在已执行测试范围内满足核心功能验证要求。该结论仅覆盖后端自动化测试、后端 smoke test、前端构建和已有截图证据，不覆盖压力、安全渗透、多浏览器、移动端、真实硬件、生产部署、后台推送、长期稳定性和生产级云端 ASR 专项。
